# Как я понял задачу из sife_plan.md

Цель: довести special-режим олимпиады до рабочего состояния под ваш процесс, не ломая обычные олимпиады.

## Что требовалось
1. Наблюдатель ищет участника (а не просто бланк), и под его QR можно выдать доп.бланк.
2. Генерация бланков и A3 должна настраиваться через редактируемые Word-файлы.
3. Нужен рабочий special-поток: импорт -> массовый допуск -> рассадка -> печать.
4. Для special-туров нужна гибкая рассадка, включая командный режим.

## Актуальный статус
Сделано:
- Наблюдатель ищет именно участников с attempt и выдает доп.бланки по выбранному участнику.
- Шаблоны бланков/A3 переведены на Word DOCX с загрузкой/скачиванием из админки.
- Массовый импорт участников + массовый допуск + ZIP по участникам (DOCX/A3/legacy PDF) работает.
- Рассадка усилена по special-правилам: 3x3 анти-соседство, аудитория капитанов, учет филиалов.
- Добавлен конструктор рассадки по столам:
  - настройки `мест за столом` и `командный режим` по каждой аудитории;
  - хранение в `special_settings.room_layouts` и `special_settings.team_room_layouts`.
- Добавлена перестройка схемы под тур:
  - `seating-plan` и `seating-plan/print` принимают `tour_number`;
  - в командном туре отображается отдельная столовая структура;
  - в админке есть кнопка печати командной схемы с выбором тура.
- A3 дефолтный шаблон обновлен под формат «книжки» (две половины листа A3 + линия сгиба).

Осталось:
- Инфраструктурный прогон тестов/сборки в текущем окружении (локально отсутствуют `pytest` и frontend toolchain `tsc/node_modules`).
